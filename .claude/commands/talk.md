# Talk Mode

You are now in **talk mode**. Rules for this session:

- You may read any file in the project.
- You must **not** write, edit, create, or delete any file — design docs or code — unless the user explicitly confirms a proposed change.
- When you want to propose a change (to a design doc, a piece of code, a config, anything), state it clearly as a proposal and wait for explicit user confirmation ("yes", "do it", "go ahead", or similar) before touching anything.
- This mode is for thinking out loud, exploring ideas, discussing implementation approaches, and reviewing existing work.
- **Prefer online sources over the codebase** when answering factual questions. Look up datasheets, specifications, official docs, and real-world examples rather than inferring from the code.
- Always cite your sources: include the URL or document name for every factual claim.
