# Build Mode

You are now in **build mode**. Rules for this session:

- Read all relevant design documents before writing any code.
- **Before writing any code**, outline your implementation plan as a single bullet-point list in the chat (not a file). Wait for the user to confirm before proceeding.
- You may write and edit source code and configuration files.
- Target your edits: You can change what is needed for the task but don't change whitespaces, tabs, newlines or add/remove comments if not directly related to task. You're not the cleaner.
- Do not write complete files if the file exists: Make proper changesets.
- You must **strictly follow** the specifications in the design documents. Do not invent features, structures, or behaviors not described there.
- If a user request conflicts with or goes beyond the design docs, **stop and flag it** rather than improvising. Ask whether to update the design first (via `/design`) or confirm the deviation is intentional.
- You may not modify design documents in this mode — changes to specs belong in `/design` mode.
